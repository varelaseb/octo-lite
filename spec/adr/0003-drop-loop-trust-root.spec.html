ADR 0003: drop the loop trust-root ceremony and independent observer

Specifications / role runtime / ADR 0003

Decisions closing the operator grill on octo-lite issue 13, 2026-07-24.

Status

Accepted.

Context

Issue 13 first built a lane provisioning trust root so delivery loops could fire: a provision-record read, a frozen OCTO_* launch environment, an environment-equals-record cross-check, and identity, entrypoint-revalidation, gates-workflow-layer, and readback gates at loop entry.

The delivery loop separately carried an independent tdd-observer role that re-ran each committed red, green, and final HEAD in an isolated worktree, plus a host-gated push after observation, an acknowledgment echo, and a live-remote pre-push readback.

ADR 0002 had made observer machinery the only red guard, stating no relocation compensates.

Operator ruling 2026-07-24 held there is literally no reason for these checks: a lane's branch and worktree are sufficient identity, and the loop should just run.

Cross-model codex review is down, so the current adversarial guard is a fresh Opus code-reviewer, refute-on-uncertainty.

Decision

The loop-entry trust root is removed wholesale: no provision-record read, no frozen-environment seam, no environment-equals-record cross-check, and no identity, entrypoint-revalidation, gates-workflow-layer, worker-claim, or readback gate; the loop derives its worktree from the process working directory and its branch from git and runs.

Host provisioning-record writing and worktree_repo cleanup ownership are retained, so the record exists for worktree provisioning and clean-abort cleanup proof rather than loop-entry identity.

The independent tdd-observer role is retired from the roster, roles.toml, the generated adapters, and the model map, and the host pushes the verified branch directly with no observer replay, acknowledgment-echo gate, or pre-push readback.

The fresh adversarial code-reviewer, refute-on-uncertainty, is the delivery-TDD guard, confirming from the committed red and green that the red genuinely failed, the green makes the named test pass, and no commit weakened, edited, or removed the bound test, while the implementer still commits spec-derived red then green.

This supersedes ADR 0002's honest-red decision that observer machinery is the only red guard and no relocation compensates: the guard relocates to the adversarial code-reviewer.

Consequences

The delivery loop collapses to its per-mode essence, and the assert-and-accept gate lattice, the launch environment seam, and the observer replay are removed from the loop and its libraries.

Delivery-TDD integrity now rests on adversarial code review rather than a mechanical observer re-run, and while codex is down that reviewer is a fresh Opus pass under a recorded operator waiver.

Mechanical replay proof is gone, so a forged red or green is caught by review judgment rather than re-execution; this is an accepted weakening per operator ruling, revisited only on observed regression.

Lane workspace identity is the process working directory and git branch, with no frozen environment or provision-record read at loop entry.
