ADR 0001: operating model boundaries

Specifications / operating model / ADR 0001

Boundary decisions closing the fresh exact-head review at PR 6 head 3a2c90edff3be3b8f2a71c82b7956aa1fc1a244e.

Status

Accepted.

Context

A fresh exact-head review of PR 6 found four executable divergences between claimed and actual behavior.

Generated Codex custom-agent files loaded with a null role, a generic model, and generic instructions in a live smoke test, so role identity never reached the Codex session.

The same generated files used relative skills.config paths that resolve only from ~/.codex/agents, missing the installed skill locations entirely.

herdr-say and herdr-drain inferred composer consumption by grepping recent pane text for the sent message, so a successful send whose transcript echoed the message stayed pasted forever.

Dead-owner recovery combined a real liveness probe with a caller-supplied boolean --operator-authorized flag, so any caller could self-assert operator authorization.

The canonical spec closed its decision chronology at Decision 102 while the audit packet already recorded Decisions 103 and 104, and no ADR existed to sign the boundary.

Five prior fix passes iterated machinery around these paths instead of removing the unreliable surface, exceeding the three-cycle return rule.

Decision

Generated adapters

Adapter generation renders Claude Markdown only; no Codex custom-agent file is generated, installed, or checked.

Install migrates a dangling or repo-owned legacy Codex custom-agent link at each of the eight exact former role names and leaves any other file at that path untouched.

OpenAI role launch

Superseded by the Decision 109 amendment below: an OpenAI-backed role pass now runs inside a Claude relay subagent, not an octo-launch pass with a worker receipt. Historical rule as originally signed: an OpenAI-backed role pass is one explicit codex exec relay launched by octo-launch, carrying exact model, effort, service tier, worktree, receipt, and the canonical role contract as the exec prompt.

The relay never discovers, references, or depends on a Codex custom agent.

The relay fails closed unless the exact effective provider, model, and effort are independently read back from the Codex session's own rollout record before mutation proceeds.

Effective identity is read from the session_meta and latest turn_context entries of the exact session's rollout file, never from the model's own self-report.

Herdr delivery proof

herdr-say and herdr-drain never read pane transcript text to infer composer state, consumption, or delivery.

A send or Enter transport attempt leaves a message pending; only the tool call's own exit status decides retry, never a text match.

Only an explicit herdr-ack from the exact recipient proves delivery for a correctness-bearing message.

Retry reuses the same message ID, presses Enter or retries transport only, and never re-pastes text that already landed.

An info-kind message stays best-effort and never claims acknowledged delivery.

Dead-owner handling

octo-control has no dead-owner recovery command and no agent-callable path asserts operator authorization.

A dead or ambiguous owner record blocks every operator-owned mutation until an explicit manual operator takeover outside role authority.

No cryptographic operator-authentication mechanism or recovery service is introduced to compensate.

Decision chronology

The canonical decision chronology in optimal-model-grill.md is authoritative through Decision 104.

The chronology is extended through Decision 108, adding the spec split, the diagram-first pages, the diagram and prose authority rule, and the loop-fire lifecycle correction.

The chronology is extended through Decision 109, migrating every worker role to a Claude Workflow subagent and retiring the octo-launch CLI pass model.

Amendment 2026-07-20

This dated amendment extends the ADR boundary through Decision 108 after the diagram-first spec split.

Decision 106 splits the operating model into one concise overview plus three canonical domain specs by independent change and review boundaries.

Decision 107 makes every canonical page diagram-first and retains prose only for exact rules, edge conditions, accessibility, and behavior a diagram cannot express.

Decision 108 makes diagrams normative for the states and relationships they encode and terse prose normative for constraints and exceptions, with one primary representation per rule.

The debate ruling-1 lifecycle correction removes the mechanical Shaped -> In Progress edge and binds loop fire to the mechanical Shaped -> Todo transition as the first delivery step.

Amendment 2026-07-20 Decision 109

Decision 109 runs every worker role as a Claude Workflow subagent of the owning session and retires the octo-launch CLI pass model.

OpenAI review roles run through a Claude relay subagent that returns codex findings verbatim with a hash the workflow binds.

Worker binding proof becomes the workflow journal plus a schema-forced acknowledgment echo the orchestrator verifies before mutation, durable TOML receipts bind only persistent sessions, and admission and revalidation gates re-home to workflow-layer helpers.

Ruling-4 records that a cross-cutting ruling propagates by a full semantic sweep of every durable surface, never a grep-only pass.

Consequences

Codex custom-agent discovery, skills.config paths, and per-role generated .toml diagnostics are removed load-bearing surfaces; only Claude Markdown adapters remain generated.

OpenAI role identity is provable from Codex's own session record instead of assumed from CLI exit status alone.

Herdr delivery status can no longer silently misreport a successful send as unsubmitted, or an unconfirmed one as submitted; a pending message stays honestly unconfirmed until acknowledged.

A dead or ambiguous owner record is a hard stop that trades availability for removing self-asserted agent authority.

The spec decision-log reference and this ADR agree, closing the conflict the review found blocking.
