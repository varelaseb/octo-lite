ADR 0002: vendored TDD skill and skill management

Specifications / delivery lifecycle / ADR 0002

Decisions closing the operator grill on octo-lite issue 11, 2026-07-23.

Status

Accepted.

Context

octo-lite carried a 51-line governance tdd skill while mattpocock/skills carries a craft tdd skill with test-quality guidance octo-lite lacked.

The octo-lite skill duplicated waiver, evidence, and prompt-TDD rules that already live canonically in the operating-model and delivery-lifecycle specs.

The upstream skill states refactoring is not part of the red-green loop, contradicting the then-current spec language spec-driven red, green, refactor.

Hand-copied third-party skills drift silently; local edits die on upstream update.

Decision

Vendored TDD skill

The octo-lite tdd skill is replaced completely by mattpocock/skills skills/engineering/tdd at a pinned upstream SHA, the blessed pristine version.

Exactly one content patch is carried: the seam gate confirms seams against the canonical spec, never the user, and spec-named seams must conform to deep-module seams; a spec whose seams are not deep-module shaped is a shaping defect bounced to shaping, never a license to invent seams.

The upstream rule that refactoring is not part of the loop is kept verbatim; the operating-model and delivery-lifecycle specs are amended to match, and refactoring arrives only through reviewer-flagged refactor-only fix passes verified green-stays-green.

The prior honest-red rule, a failing test is not red when it fails for setup, syntax, or dependency reasons, is deliberately dropped by operator ruling; observer machinery is the only red guard, and no relocation compensates.

Waiver, evidence, and prompt-TDD rules are not restored to the skill; the specs remain their sole canonical home.

Skill management

skillfile owns all skill installation and management: one committed Skillfile lists the vendored upstream tdd entry and local entries for octo-lite's own skills, Skillfile.lock pins exact upstream revisions, and skillfile pin preserves the seam-gate patch across upstream updates.

install-path targets place managed skills into agents/skills, and platform install lines place a copy or symlink into .claude/skills and .codex/skills on every install.

The npx skills installer is rejected: for this use it is a strict subset of skillfile with no lockfile and no patch preservation, and adopting both would split ownership of the same files.

The agents directory contract is amended: agents/skills is a managed skillfile install target, while the rest of agents remains generated adapters only.

scripts/install-octo-lite --check verifies the mirror invariant: every managed skill present in agents/skills resolves identically from .claude/skills and .codex/skills.

Consequences

Implementers gain upstream test-craft guidance, seams, anti-patterns, and vertical slicing, at the cost of octo-lite-specific prose the specs already carry.

A red that fails for an unintended reason is no longer textually forbidden; this is an accepted weakening, revisit only on observed regression.

Upstream skill updates become skillfile install --update with the patch re-applied, replacing manual re-copying.

Standalone cleanup waits for a reviewer flag; refactor latency is bounded by review cadence rather than implementer discretion.
