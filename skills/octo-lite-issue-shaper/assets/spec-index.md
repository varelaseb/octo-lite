# Spec Index

Canonical specs are organized by durable product or system area. Linear and
GitHub issue IDs belong in references and decision logs, not canonical
filenames.

Use this index when deciding where durable behavior belongs. `spec/` and
`spec/adr/` are source-of-truth surfaces for agents and reviewers.

Follow the target repo's `AGENTS.md` format signal. Repos without
`Spec format: spec-chat`, or with `Spec format: markdown`, keep this Markdown
index as the canonical navigation surface. A spec-chat repo uses the same
domain and ADR structure in `index.spec.html`, with anchored sections, one
sentence per prose line, and valid internal links; do not keep a Markdown
counterpart.
