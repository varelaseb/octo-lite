# Spec Index

Canonical specs are organized by durable product or system area. GitHub issue
IDs belong in references and decision logs, not canonical filenames.

Use this index when deciding where durable behavior belongs. `spec/` and
`spec/adr/` are source-of-truth surfaces for agents and reviewers.
